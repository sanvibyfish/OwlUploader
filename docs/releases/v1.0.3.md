# v1.0.3 Release Notes

**Release Date**: 2026-03-20

## New Features

### Alibaba Cloud OSS Support

Added multi-cloud support — OwlUploader now works with both Cloudflare R2 and Alibaba Cloud OSS.

- New `Cloud Provider` selector when adding accounts (R2 / OSS)
- OSS Region picker with all major Chinese and international regions
- Automatic endpoint generation based on selected region
- Full feature parity: browse, upload, download, rename, move, delete, copy link

**Technical approach**: OSS is S3-compatible, so we reuse the existing AWS S3 SDK with provider-aware configuration (virtual-hosted-style addressing for OSS, path-style for R2).

## Bug Fixes

### OSS: SecondLevelDomainForbidden on Connect

OSS requires virtual-hosted-style addressing (`bucket.endpoint/key`), but the S3 SDK defaulted to path-style. Added `forcePathStyle` configuration per provider.

### OSS: Copy Link URL Format

Generated URLs used path-style format for OSS accounts. Introduced `buildStorageURL()` to produce correct virtual-hosted-style URLs (`bucket.host/key`).

### OSS: Folder Rename Failure

Renaming folders failed because: (1) OSS doesn't support `CopyObject` on zero-byte folder markers, (2) only the folder marker was copied, not its contents. Fixed by implementing batch copy-then-delete with `PutObject` for folder markers.

### OSS: False CDN Purge Failure after Upload

Upload queue unconditionally collected CDN purge URLs. For OSS accounts (which don't support Cloudflare CDN purge), this caused false "partial failure" reports. CDN purge collection is now gated behind `supportsCDNPurge`.

### Keychain Identifier Desync on Account Edit

Editing `Account ID` or `Access Key ID` changed the Keychain lookup key, orphaning stored credentials. `keychainAccountIdentifier` now uses the immutable account UUID.

### EditAccountSheet: Silent Credential Load Failure

Loading stored credentials used `try?`, silently failing without user notification. Now shows an orange warning when Keychain read fails.

### EditAccountSheet: Silent Token Delete Failure

Deleting Cloudflare API Token used `try?`, swallowing errors. Now propagates errors to the user.

## Improvements

### Loading State: Reference-Counted isLoading

Replaced the single `isLoading: Bool` flag with a `loadingCount: Int` counter. Multiple concurrent operations no longer overwrite each other's loading state.

### Folder Operation Mutex

`renameFolder` and `moveFolder` now check `isFolderOperationInProgress` to prevent overlapping execution. `CancellationError` is properly propagated instead of being swallowed.

### AddAccountSheet: Task Cancellation on Dismiss

Clicking "Cancel" during a connection test now cancels the background network request.

### Localization

Replaced hardcoded strings in AddAccountSheet:
- `"Cloudflare Account"` section header
- `"留空则自动生成..."` OSS endpoint hint

### OSS Region Validation

`ossRegion` field now trims whitespace before validation, preventing empty-space bypass.
