# v1.0.2 Release Notes

**Release Date**: 2026-03-18

## Performance

### File List Sorting Cache

Fixed a performance issue where `filterAndSort()` was recomputed on every SwiftUI view render, even for unrelated state changes (drag hover, sheet toggle, etc.).

- Sorting results are now cached in `@State` and only recomputed when the 5 relevant inputs change (`fileObjects`, `searchText`, `filterType`, `sortOrder`, `sortAscending`)
- Eliminates redundant O(n log n) sort operations during unrelated UI interactions

## Bug Fixes

### Public Domains Not Saving

Fixed a bug where public domain settings would not take effect after saving.

**Root Cause (1)**: When editing an account without changing the secret key, `R2Service.currentAccount` (a struct copy) was not updated ŌĆö URL generation continued using stale data.

**Root Cause (2)**: Typing a domain in the text field and clicking "Save" directly (without pressing Enter or "+") would lose the input, because the value remained in `newDomain` and was never appended to the `publicDomains` array.

**Fix**:
- Added `R2Service.updateCurrentAccount(_:)` for lightweight account sync without rebuilding the S3 client
- `saveAccount()` now auto-harvests pending input from the domain text field before saving
- Both `EditAccountSheet` and `AddAccountSheet` are fixed

### Domain Input Field Label

Fixed the domain input field displaying `cdn.example.com` as a left-side label instead of placeholder text. Changed from `TextField(title, text:)` to `TextField("", text:, prompt:)` for correct macOS Form behavior.

## New Features

### Multi-Domain Copy Link

When multiple public domains are configured, the "Copy Link" action now shows a submenu/dropdown to let users choose which domain to use.

- **Right-click menu** (Table & Grid view): Single domain ŌåÆ direct copy; multiple domains ŌåÆ submenu with each domain as an option
- **Preview toolbar**: Single domain ŌåÆ button; multiple domains ŌåÆ dropdown menu
- Follows the Cyberduck submenu pattern, consistent with industry conventions
